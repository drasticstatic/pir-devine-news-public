# Contributing

Thank you for your interest in Pir Devine News!

## How to Contribute

### Reporting Issues

Open a GitHub Issue with:
- A clear description of the problem
- Steps to reproduce (if a bug)
- Expected vs. actual behavior

### Submitting Changes

1. **Fork** the repository
2. **Create a branch** for your change (`git checkout -b feature/your-change`)
3. **Commit** with a clear, descriptive message
4. **Open a Pull Request** describing what changed and why

### Content Contributions

For editorial or content contributions:
- Keep content factual and sourced
- Follow the existing format conventions in `data/` and `dashboard/`
- One logical change per PR

## Security Issues

Do **not** report security vulnerabilities in public Issues. See [SECURITY.md](https://github.com/drasticstatic/.github/blob/main/SECURITY.md) for responsible disclosure.
