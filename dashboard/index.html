How the Devine News Hub Works

Think of this system as an automated assistant that does the filing for us — so the committee can focus on the content, not the coordination.

For Members — Submitting

  • Click Submit Work to share literature, artwork, or both.
  • Your piece goes directly into the committee's Google Drive, correctly named and filed.
  • You'll receive a Thank You email immediately with our current meeting schedule.

For Committee — Reviewing

  • Every hour (or on demand), the sync bot checks Drive for new submissions.
  • Each new piece appears here with a status badge — Incoming → Needs Review → Approved → Published.
  • Click Open in Drive on any submission to edit it in Google Docs.
  • Art thumbnails appear in the gallery above for quick visual review.

Do I Need to Know GitHub?

No. Members only need to use this portal and their regular Gmail/Google Drive. The technical automation runs quietly in the background.

What if a Submission Doesn't Appear?

The sync runs every hour. If you're in a rush, contact the Tech Chair to trigger a manual sync from the GitHub Actions tab.

gws CLI Setup Guide

The gws CLI connects the automation scripts to your Google Workspace account. Full step-by-step instructions are in setup/GWS_SETUP.md in the private repo. Here's the condensed version:

1 — Google Cloud Console

  • Create a project → Enable Drive API + Gmail API
  • Credentials → OAuth 2.0 Client ID → Desktop app type
  • Download JSON → save as ~/.config/gws/client_secret.json

2 — Authenticate

gws auth login

Browser opens → sign in as pir.devine.news@gmail.com → grant Drive + Gmail access. Token stays local, never committed.

3 — Create the private config

Create data/committee/config.env (gitignored):

SUBMISSIONS_FOLDER_ID=your_folder_id_here
TEMPLATES_FOLDER_ID=your_folder_id_here
APPROVED_FOLDER_ID=your_folder_id_here
GOOGLE_ACCOUNT=pir.devine.news@gmail.com

Find a folder ID in the Drive URL after /folders/.

4 — Add the Drive URL to data.json

Copy the Submissions folder URL → add to dashboard/data.json → driveUrl. The Drive panel activates on the next push.

5 — Run the sync

./scripts/gws-sync.sh
