🌀 Welcome

How the Devine News Hub Works

Think of this system as an automated assistant that does the filing for us — so the committee can focus on the content, not the coordination.

For Members — Submitting

  • Click Submit Work to share literature, artwork, or both.
  • Your piece goes directly into the committee's Google Drive, correctly named and filed.
  • You'll receive a Thank You email immediately with our current meeting schedule.

For Committee — Reviewing

  • Every hour (or on demand), the sync bot checks Drive for new submissions.
  • Each new piece appears here with a status badge — Incoming → Needs Review → Approved → Published.
  • Click Open in Drive on any submission to edit it in Google Docs.
  • Art thumbnails appear in the gallery above for quick visual review.

Do I Need to Know GitHub?

No. Members only need to use this portal and their regular Gmail/Google Drive. The technical automation runs quietly in the background.

What if a Submission Doesn't Appear?

The sync runs every hour. If you're in a rush, contact the Tech Chair to trigger a manual sync from the GitHub Actions tab.

🌿 Welcome to the Portal

What Is This?

This is the De Vine News Committee Portal — an interactive space where PIR® members and the newsletter committee come together each month.

Here you can:

  • Read past and current newsletter editions
  • Submit your writing, poetry, artwork, or reflections for an upcoming issue
  • See open service positions and get involved with the PR or Literature Committee
  • Track themes — each issue is centered on a spiritual principle from the 12 Steps

Behind the scenes, the committee uses this portal to review, approve, and publish submissions — all through a secure, automated workflow that keeps unpublished content private.

Built and maintained as committee service by the PIR® Tech & PR Committees. No special expertise required to submit — only a willingness to share.

🔧 For the technically curious

Public repo (pir-devine-news-public)
Static dashboard and newsletter archive for the De Vine News committee portal. Served via GitHub Pages. Receives filtered content automatically synced from the private workflow repo on every push to main. No credentials, no raw data, no unpublished submissions.

Private repo (pir-devine-news)
Full newsletter workflow hub for the Psychedelics In Recovery (PIR®) 12-Step Fellowship newsletter committee. Includes automation scripts, committee data, Google Drive integration via GWS CLI, GitHub Actions sync pipeline, and the full source for all public-facing dashboard content.

💌 A Message from the Board

Welcome to de Vine, Psychedelics in Recovery's newsletter. I'm grateful you're here.

PIR was born from a simple but difficult question: How do we talk honestly about recovery in a world where psychedelics are increasingly visible, medicalized, politicized, and misunderstood — without losing our integrity or one another? Over the years, our fellowship has grown into a diverse, sometimes imperfect, and deeply committed community wrestling with that question together.

This newsletter is an extension of that effort. de Vine is not a platform for promotion, persuasion, or dogma. It is a space for reflection, accountability, service, and shared learning. You'll find reports from service bodies, creative work emerging from the fellowship, and thoughtful engagement with recovery, integration, and the responsibilities that come with both.

Growth brings complexity. As PIR expands across geographies, modalities, and viewpoints, we are continually challenged to balance openness with safety, innovation with humility, and individual experience with collective care. These tensions are not signs of failure; they are signs that something alive is being tended. Our task is not to eliminate disagreement, but to remain grounded in respect, transparency, and a shared commitment to recovery.

Whether you are new to PIR or have been walking with us for years, I invite you to see this publication as a conversation rather than a conclusion. Read closely. Question thoughtfully. Serve where you are able. And remember that none of us carries this work alone.

Thank you to the PR Committee and all contributors who brought this first issue into being. I look forward to what we will continue to grow — together.

In service,
Kevin Franciotti
Co-founder, PIR® Board of Directors

⚙️ How the pipeline works

Sync & Submissions

Submission flow

  1. A member fills out the Submit page and clicks Send.
  2. The form POST hits the Apps Script web app running on the pir.devine.news@gmail.com Google account.
  3. Apps Script writes a row to the Submissions Google Sheet and sends a confirmation email to the member and a notification to the committee inbox.
  4. Apps Script calls the GitHub API to update dashboard/data.json in the public repo with the new submission entry.
  5. GitHub Pages serves the updated data.json — the submission appears on this dashboard after the next page refresh. GitHub Pages CDN can take up to a few minutes to propagate — if cards don't appear right away, wait a moment and use ↺ Refresh.

Removing a submission

  1. Click 🔒 Manage next to this button and enter the committee admin passphrase.
  2. Each submission card gains a ✕ Remove button.
  3. Clicking Remove sends a password-authenticated request to the Apps Script, which removes the entry from data.json and marks the row Archived in the Google Sheet.
  4. The card disappears from the dashboard immediately.

Updating submission status

  1. Open the Submissions Sheet ↗ and edit the Status column for any row.
  2. Valid values: Incoming · Needs Review · Approved · Published · Archived
  3. Click 🔒 Manage → ↕ Sync from Sheet on this page — the dashboard updates to reflect the new statuses.

The ↺ Refresh button

Re-fetches data.json from GitHub Pages on demand. Use this to pull in any submissions that arrived since you opened the page, or after an Apps Script re-deploy.

Code sync (private → public repo)

Every push to main on the private repo triggers a GitHub Actions workflow that strips private content (scripts, setup docs, committee data) and force-pushes the filtered result to pir-devine-news-public. data.json is excluded from this sync so live submission data is never overwritten.
