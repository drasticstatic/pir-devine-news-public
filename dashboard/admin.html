🌀 Holacratic Publishing Governance — The Vision

Instead of one person controlling what goes live, publishing becomes a lightweight consent process aligned with PIR's holacratic values. The admin agent drafts, the committee reviews, and the site updates itself if no one objects.

📝 Agent stages draft → 🔗 Preview URL generated → 📧 Email to all editors → 🗳 Voting window opens → ✅ No objection → auto-publish → 📚 Sidenav updates live

⚙️ Architecture

How It Works

1
Committee member types a request in plain English — no commands, no file paths.
2
Admin Agent reads current files — submission data, edition pages, form settings, sidenav.
3
Agent proposes the change showing a diff — "Here's exactly what I'd update. Approve?"
4
Editor confirms — one click or "yes" in the chat.
5
Agent commits and pushes — site updates in minutes. Every change has a full Git history and can be reverted instantly.

Two-Agent Architecture

🤖
Admin Agent — Site Management

Manages deadlines, themes, submissions, editions, and bulletin. Access limited to dashboard/ files. Never reads credentials or member emails.

✨
Inner Voice Agent — Writing Facilitation

Helps members write complete, authentic submissions through Socratic dialogue. Separate system prompt, separate session, no site access. Facilitator — never ghostwriter.

Future: skill-based sub-agents delegated by coordinator agents — Claude Haiku for routine queries, Sonnet for complex drafting. Cost-efficient by design.
