🌀 Holacratic Publishing Governance — The Vision

Instead of one person controlling what goes live, publishing becomes a lightweight consent process aligned with PIR's holacratic values. The admin agent drafts, the committee reviews, and the site updates itself if no one objects.

⚙️ Architecture

How It Works

1
Committee member types a request in plain English — no commands, no file paths.
2
Admin Agent reads current files — submission data, edition pages, form settings, sidenav.
3
Agent proposes the change showing a diff — "Here's exactly what I'd update. Approve?"
4
Editor confirms — one click or "yes" in the chat.
5
Agent commits and pushes — site updates in minutes. Every change has a full Git history and can be reverted instantly.

Two-Agent Architecture

🤖
Admin Agent — Site Management

Manages deadlines, themes, submissions, editions, and bulletin. Access limited to dashboard/ files. Never reads credentials or member emails.

✨
Inner Voice Agent — Writing Facilitation

Helps members write complete, authentic submissions through Socratic dialogue. Separate system prompt, separate session, no site access. Facilitator — never ghostwriter.

Future: skill-based sub-agents delegated by coordinator agents — Claude Haiku for routine queries, Sonnet for complex drafting. Cost-efficient by design.

📧 Drive Sync Access List

Committee Access — Two Tiers

📋 You have pending changes — click 📝 Propose Change via Agent in the footer to apply them.
All changes are proposals — the agent shows a diff and waits for your approval. Removing an address does not revoke existing Drive permissions retroactively.
✏️ Tier 1 — Editors (Writer access on Google Drive)

Lives in data/committee/emails.txt (private). Every gws-sync.sh run grants these editors Writer access on the pir.devine.news Drive folder and all Approved/ review copies. Also used for holacratic edition preview notifications.

👁️ Tier 2 — Reviewers (Commenter access — read + comment only)

Lives in data/committee/reviewers.txt (private). These members can read and comment on Drive content but cannot edit. Add advisors, readers, or guests who should follow along without editing access.

Removing an address stops future Drive access grants — existing permissions are not revoked retroactively.
📊 Phase 4 Build Status

What You're Looking At

This is the De Vine News AI Admin Dashboard — a newsletter management interface for the PIR fellowship committee. It's designed to let any committee member manage submissions, deadlines, themes, and publishing through plain-English conversation. No code, no GitHub, no CLI required.

✅ Fully live right now
  • Public demo — anyone can explore using passphrase pirdemo
  • Real auth gate — the actual committee passphrase calls Google Apps Script and verifies identity before granting admin access
  • Committee email management — two-tier Drive access lists (Editors + Reviewers) manageable from this dashboard
  • Google Drive sync — gws-sync.sh pulls real submission data from Drive into the committee portal
  • Inner Voice Agent UX — conversational writing facilitator for member submissions
  • Submission form, newsletter archive, bulletin board, resources page
  • Private→public auto-publish pipeline via GitHub Actions
🔨 Chat: UX live, AI responses simulated

The chat interface, the chip ticker, the approve/cancel diff preview, the typing indicator — all fully functional. Responses are currently powered by JavaScript pattern-matching, not a real AI call. This means the UX is exactly what Phase 4 will feel like — the intelligence just needs to be wired in.

What's missing: a lightweight Apps Script endpoint that proxies messages to the Claude API, reads the current file state as context, and returns real agent responses. Estimated build: ~2–3 hours. The passphrase gate, the UI, and the diff-approval workflow are already built and waiting for it.

📋 Next to build
  • API proxy — Apps Script endpoint forwarding chat to Claude Sonnet with live file context
  • Holacratic publish flow — agent stages a draft, emails all editors with a preview link, auto-publishes if no objection arrives in 72 hours
  • Scheduled Drive sync — hourly GitHub Actions cron pulling new submissions automatically
  • Deadline countdown widget — live days-remaining display on portal and submission form
  • Google OAuth login — portal access limited to whitelisted Gmail accounts
Stack: GitHub Pages (hosting) · Google Apps Script (form backend + auth) · Google Drive (file storage) · Claude API (coming) · GitHub Actions (auto-publish) · All free at current scale.
Security: No credentials, committee emails, or private Drive IDs ever reach the public repo. The demo passphrase is intentionally public. The real admin passphrase lives in Apps Script Script Properties, not in any file.
